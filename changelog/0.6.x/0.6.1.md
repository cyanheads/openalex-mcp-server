# 0.6.1 — 2026-04-24

Picks up `@cyanheads/mcp-ts-core` 0.7.0 and the skill/script updates that ship with it. No behavior changes in this server's handlers — this is a maintenance release.

## Changed

- Bumped `@cyanheads/mcp-ts-core` `^0.6.17` → `^0.7.0` (no breaking changes).
- `CLAUDE.md` / `AGENTS.md` — removed the stale `.toLocaleString()` caveat in the tool-pattern comment and Checklist. Framework 0.7.0's `format-parity` linter is now locale-aware: digit-group separators (commas, periods, spaces, narrow no-break, Arabic thousands) are stripped before sentinel matching, so `.toLocaleString()` is safe on linted numeric fields. Compact/scientific forms still fail by construction.
- Resynced framework skills: `api-linter` (1.0 → 1.1 — `describe-on-fields` recursion table), `maintenance` (1.4 → 1.5 — direct framework-update adoption, tiered cost/benefit for third-party), `release-and-publish` (2.0 → 2.1 — transient-failure retry protocol), `report-issue-framework` (1.2 → 1.3), `report-issue-local` (1.2 → 1.3). Label restructuring in the issue skills splits primary (required, pick one) from secondary (optional, stackable) — matches the new `regression`/`performance`/`security`/`breaking-change` labels in the framework repo.
- Resynced framework scripts: `scripts/build-changelog.ts` and `scripts/devcheck.ts` — picks up [#51](https://github.com/cyanheads/mcp-ts-core/issues/51) fix for the devcheck crash on single-file `CHANGELOG.md` projects. No impact on this server (which uses the directory-based changelog), but keeps the scripts content-identical to upstream.
